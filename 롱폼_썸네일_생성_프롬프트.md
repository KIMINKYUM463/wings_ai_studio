# 롱폼 썸네일 생성 기능 - API 및 프롬프트 가이드

## 📋 개요
유튜브 롱폼 콘텐츠용 썸네일을 AI로 생성하는 기능입니다. 주제, 커스텀 문구, 벤치마킹 썸네일 분석 등을 활용하여 클릭률이 높은 썸네일을 생성합니다.

---

## 🔌 API 정보

### 기본 정보
- **이미지 생성**: Replicate API (`google/nano-banana-pro`)
- **벤치마킹 분석**: OpenAI GPT-4o (Vision)
- **HTTP 메서드**: `POST`
- **인증 방식**: 
  - Replicate: Token (Authorization 헤더)
  - OpenAI: Bearer Token (Authorization 헤더)

---

## 📝 함수 시그니처

### generateAIThumbnail

```typescript
export async function generateAIThumbnail(
  topic: string,                    // 주제 (필수)
  replicateApiKey: string,          // Replicate API 키 (필수)
  imageStyle?: string,              // 이미지 스타일 (선택)
  customText?: string,              // 커스텀 문구 (선택)
  customStylePrompt?: string,      // 커스텀 스타일 프롬프트 (선택)
  characterDescription?: string,    // 캐릭터 설명 (선택)
  withoutText?: boolean,            // 텍스트 없이 생성 여부 (선택)
  benchmarkThumbnailUrl?: string,   // 벤치마킹 썸네일 URL (선택)
  openaiApiKey?: string,            // OpenAI API 키 (벤치마킹 분석용, 선택)
  analyzedBenchmarkStyle?: string  // 사전 분석된 벤치마킹 스타일 (선택)
): Promise<string>
```

**반환값**: 생성된 썸네일 이미지 URL

## 🎨 이미지 스타일 매핑

### imageStyle 파라미터에 따른 스타일 프롬프트

| imageStyle | 스타일 프롬프트 |
|-----------|----------------|
| `stickman-animation` | `stickman animation style, 2D vector cartoon, white circular face, simple black outline, dot eyes, curved mouth, thin black limbs, vibrant colors, flat cel shading, thick bold outlines, solid color fills` |
| `realistic` 또는 `realistic2` | `hyperrealistic, photorealistic masterpiece, 8K, ultra-detailed, sharp focus, cinematic lighting, shot on a professional DSLR camera with a 50mm lens` |
| `animation2` | `flat 2D vector illustration, minimal vector art, stylized cartoon character, thick bold black outlines, unshaded, flat solid colors, cel-shaded, simple line art, comic book inking style, completely flat, no shadows, no gradients, no depth` |
| `animation3` | `European graphic novel style, bande dessinée aesthetic, highly detailed traditional illustration, hand-drawn ink lines with cross-hatching shadows, sophisticated and muted color palette, atmospheric, cinematic frame` |

### thumbnailStyle 파라미터 (별도 설정)

| thumbnailStyle | 스타일 프롬프트 |
|---------------|----------------|
| `realistic` | `hyperrealistic, photorealistic masterpiece, 8K, ultra-detailed, sharp focus, cinematic lighting, shot on a professional DSLR camera with a 50mm lens, realistic textures, natural lighting, professional photography` |
| `animation` | `2D animation style, cartoon illustration, vibrant colors, stylized art, flat design, cel-shaded, animated character design, colorful and playful, animated movie style, Disney animation style` |

**참고**: `thumbnailStyle`이 설정되지 않으면 `imageStyle`을 사용합니다.

## 🔍 벤치마킹 썸네일 분석

### analyzeThumbnailStyle 함수

벤치마킹 썸네일 이미지를 분석하여 스타일 정보를 추출합니다.

#### 함수 시그니처

```typescript
async function analyzeThumbnailStyle(
  thumbnailImageUrl: string,  // 벤치마킹 썸네일 이미지 URL
  openaiApiKey: string       // OpenAI API 키
): Promise<string>
```

#### 분석 프롬프트 (OpenAI GPT-4o Vision)

```
이 유튜브 썸네일 이미지를 정밀하게 분석하여 다음 정보를 추출해주세요:

⚠️ 매우 중요: 텍스트 위치와 레이아웃을 정확히 분석해야 합니다.

1. 텍스트 위치 및 레이아웃 (가장 중요)
   - 텍스트가 썸네일의 어느 위치에 있는지 (상단, 중앙, 하단, 좌측, 우측 등)
   - 텍스트 정렬 방식 (중앙 정렬, 좌측 정렬, 우측 정렬)
   - 텍스트가 몇 줄인지, 각 줄의 위치
   - 텍스트와 이미지의 비율 (텍스트가 차지하는 영역)
   - 텍스트 주변 여백과 간격

2. 텍스트 디자인 및 효과
   - 텍스트 색상 (주요 색상, 대비 색상)
   - 텍스트 효과 (그림자, 외곽선, 배경 박스, 그라데이션 등)
   - 폰트 스타일 (굵기, 크기, 폰트 종류 추정)
   - 텍스트의 가독성을 위한 배경 처리 (반투명 박스, 블러 효과 등)

3. 색상 스타일
   - 주요 색상 팔레트
   - 배경 색상
   - 강조 색상
   - 전체적인 색상 톤 (밝은, 어두운, 따뜻한, 차가운 등)

4. 레이아웃 구성
   - 이미지와 텍스트의 배치 비율
   - 주요 시각적 요소의 위치
   - 전체적인 구성 (대칭, 비대칭, 균형 등)

5. 시각적 스타일
   - 이미지/일러스트레이션 스타일
   - 전체적인 분위기와 톤

분석 결과를 영어로 작성하고, 특히 텍스트 위치와 레이아웃을 정확하게 설명해주세요.
```

## 🖼️ 썸네일 생성 프롬프트

### 1. 기본 프롬프트 (벤치마킹 없을 때)

```
YouTube thumbnail for video about: {topic}. High quality, eye-catching, professional thumbnail design. Bright colors, clear text area, engaging composition. 16:9 aspect ratio.
```

커스텀 문구가 있는 경우:
```
YouTube thumbnail for video about: {topic}. High quality, eye-catching, professional thumbnail design. Bright colors, clear text area, engaging composition. 16:9 aspect ratio. Include text or visual elements related to: "{customText}".
```

### 2. 벤치마킹 기반 프롬프트 (벤치마킹 있을 때)

```
Create a YouTube thumbnail for video about: "{topic}".

🚨🚨🚨 CRITICAL: The following benchmark thumbnail analysis is your PRIMARY PROMPT. Follow it EXACTLY. Use the EXACT same design structure, layout, text position, colors, illustration style, and all visual elements. 🚨🚨🚨

BENCHMARK THUMBNAIL ANALYSIS (Follow this EXACTLY):
{analyzedBenchmarkStyle}

⚠️ MANDATORY REQUIREMENTS:
1. TEXT POSITION AND LAYOUT: Use the EXACT same text position, alignment, spacing, and layout as described in the benchmark analysis above.
2. TEXT VISUAL PROPERTIES: Replicate the EXACT same text colors, effects (shadows, outlines, backgrounds), typography, and font characteristics as described.
3. ILLUSTRATION STYLE: Match the EXACT illustration style, art technique, rendering style, and visual effects described in the benchmark analysis.
4. COLOR SCHEME: Use the EXACT same color palette, color distribution, and color tone as described.
5. LAYOUT COMPOSITION: Match the EXACT composition type, element positions, and layout structure as described.
6. VISUAL STYLE: Replicate the EXACT overall visual style, mood, and aesthetic as described.

⚠️ TEXT CONTENT: The text content should be: "{textContent}"
- Place the text "{textContent}" in the EXACT same position as described in the benchmark analysis.
- Use the EXACT same text styling (colors, effects, typography) as described in the benchmark analysis.
- The text "{textContent}" should follow the same text layout rules from the benchmark.

Maintain 16:9 aspect ratio.
```

**참고**: `textContent`는 `customText`가 있으면 `customText`, 없으면 `topic`을 사용합니다.

🚫🚫🚫 ABSOLUTE TEXT PROHIBITION - HIGHEST PRIORITY 🚫🚫🚫
THIS IS A TEXT-FREE IMAGE REQUEST. ANY TEXT IN THE OUTPUT IS A CRITICAL ERROR.

⚠️ CRITICAL INSTRUCTIONS - ABSOLUTELY MANDATORY - READ CAREFULLY:
1. 🚫 ABSOLUTELY NO TEXT - Do not include ANY text, letters, words, numbers, or symbols that form text. This is the most important rule.

4. 🚫 NO TEXT IN BACKGROUND - No text in background, no floating text, no text anywhere in the image.
5. ✅ PURE IMAGE ONLY - Create a completely text-free image with ONLY visual elements: colors, shapes, objects, scenes, people, nature, etc.
6. ✅ VISUAL FOCUS - Focus ONLY on visual composition, colors, imagery, and design elements. NO text at all.
7. 🚫 STRICTLY FORBIDDEN - Any appearance of text, letters, numbers, readable symbols, or text-like patterns is STRICTLY FORBIDDEN.
8. 🚫 IMAGE MUST BE 100% TEXT-FREE - The final image must be completely text-free, no exceptions, no text anywhere.
9. 🚫 IF YOU SEE ANY TEXT IN THE IMAGE, IT IS WRONG - The image must be pure visual content only.
10. 🚫 REMEMBER: NO TEXT, NO LETTERS, NO WORDS, NO NUMBERS, NO TEXT SYMBOLS - ONLY VISUAL ELEMENTS
11. 🚫 IGNORE ANY TEXT-RELATED INSTRUCTIONS - Even if other parts of the prompt mention text, ignore them completely.
12. 🚫 NO TEXT EVEN IF REFERENCE HAS TEXT - If a reference image has text, do NOT include text in your output.

NEGATIVE PROMPT: text, letters, words, typography, English text, Korean text, Chinese text, Japanese text, numbers, logos, watermarks, signs, labels, captions, subtitles, written text, text overlay, any text elements, any written content, any letters, any numbers, any symbols that form text, text on image, text in background, floating text, alphabet, characters, fonts, typeface, lettering, inscription, writing, script, calligraphy, text box, text area, text banner, text label, text sign, text poster, text display, text graphics, text design, text art, text illustration, any form of text, any form of writing, any form of letters, any form of numbers, any readable text, any visible text, any text content

## 🔌 Replicate API 요청

### 엔드포인트

```
POST https://api.replicate.com/v1/models/google/nano-banana-pro/predictions
```

### 요청 헤더

```json
{
  "Authorization": "Token {replicateApiKey}",
  "Content-Type": "application/json"
}
```

### 요청 Body

```json
{
  "input": {
    "prompt": "{생성된 프롬프트}",
    "aspect_ratio": "16:9",
    "output_format": "png",
    "negative_prompt": "{negativePrompt}"  // withoutText가 true일 때만 포함
  }
}
```

### 응답 처리 (폴링)

1. **초기 응답**: `status: "processing"` 또는 `status: "starting"`
2. **폴링**: 5초마다 상태 확인 (최대 60회, 약 5분)
3. **완료**: `status: "succeeded"` → `output`에서 이미지 URL 추출
4. **실패**: `status: "failed"` → 에러 메시지 반환

### 응답 형식

```json
{
  "id": "prediction_id",
  "status": "succeeded",
  "output": "https://replicate.delivery/..."
}
```

**참고**: `output`은 문자열 또는 배열일 수 있습니다. 배열인 경우 첫 번째 요소를 사용합니다.

## 📋 구현 가이드

### 전체 프로세스

```typescript
// 1. 벤치마킹 썸네일 분석 (선택사항)
let analyzedBenchmarkStyle: string | undefined
if (benchmarkThumbnailUrl && openaiApiKey) {
  analyzedBenchmarkStyle = await analyzeThumbnailStyle(
    benchmarkThumbnailUrl,
    openaiApiKey
  )
}

// 2. 썸네일 생성
const thumbnailUrl = await generateAIThumbnail(
  topic,
  replicateApiKey,
  imageStyle,
  customText,
  customStylePrompt,
  characterDescription,
  withoutText,
  benchmarkThumbnailUrl,
  openaiApiKey,
  analyzedBenchmarkStyle
)
```

### 프롬프트 생성 우선순위

1. **analyzedBenchmarkStyle이 있는 경우**: 벤치마킹 기반 프롬프트 사용 (최우선)
2. **benchmarkStylePrompt가 있는 경우**: 벤치마킹 기반 프롬프트 사용
3. **기본 프롬프트**: 벤치마킹 없을 때 사용

**중요**: `analyzedBenchmarkStyle`이 있으면 무조건 벤치마킹 기반 프롬프트를 사용합니다. 다른 로직은 무시됩니다.

## ⚠️ 주의사항

### 1. 벤치마킹 분석 결과 우선순위

- `analyzedBenchmarkStyle` 파라미터가 제공되면 **무조건** 벤치마킹 기반 프롬프트를 사용합니다.
- `analyzedBenchmarkStyle`이 없고 `benchmarkThumbnailUrl`이 있으면 실시간으로 분석합니다.
- 분석 실패 시에도 기본 프롬프트로 계속 진행합니다.

### 2. 텍스트 없이 생성 모드

- `withoutText = true`일 때는 프롬프트 맨 앞에 텍스트 금지 키워드를 추가합니다.
- Negative prompt도 함께 사용하여 텍스트 생성을 최대한 방지합니다.
- 벤치마킹 스타일이 있어도 텍스트 관련 부분은 제거됩니다.

### 3. 스타일 프롬프트 우선순위

1. `thumbnailStyle`이 설정된 경우: `thumbnailStyle` 프롬프트 사용
2. `thumbnailStyle`이 없고 `imageStyle`이 있는 경우: `imageStyle` 프롬프트 사용
3. `customStylePrompt`가 있는 경우: `customStylePrompt` 사용

### 4. API 키 요구사항

- **Replicate API 키**: 필수 (썸네일 생성)
- **OpenAI API 키**: 벤치마킹 분석 시에만 필요
