# Cloud Run Concurrency 설명

## Concurrency의 의미

**Concurrency = 인스턴스당 동시 요청 수**

- **Concurrency 1**: 각 인스턴스가 한 번에 1개 요청만 처리
- **Concurrency 80**: 각 인스턴스가 한 번에 최대 80개 요청 처리

## 실제 동작 방식

### 현재 설정
- **메모리**: 4GB
- **CPU**: 2코어
- **최대 인스턴스**: 30개
- **Concurrency**: 1 (또는 80)

### Concurrency 1인 경우

```
A수강생 요청 → 인스턴스 1에서 처리 (5-60분 소요)
B수강생 요청 → 인스턴스 2에서 처리 (동시에 시작!) ✅
C수강생 요청 → 인스턴스 3에서 처리 (동시에 시작!) ✅
...
30번째 수강생 요청 → 인스턴스 30에서 처리 (동시에 시작!) ✅
31번째 수강생 요청 → 대기... (인스턴스가 추가로 생성되거나 기존 요청이 끝나면 처리)
```

**결론**: 최대 30개 요청을 동시에 처리 가능!

### Concurrency 80인 경우 (현재 설정)

```
A수강생 요청 → 인스턴스 1에서 처리
B수강생 요청 → 인스턴스 1에서 처리 (같은 인스턴스!)
C수강생 요청 → 인스턴스 1에서 처리 (같은 인스턴스!)
...
80번째 수강생 요청 → 인스턴스 1에서 처리 (같은 인스턴스!)
81번째 수강생 요청 → 인스턴스 2에서 처리
...
최대 30개 인스턴스 × 80개 요청 = 2,400개 요청 동시 처리 가능
```

## 타임아웃 걱정?

### 타임아웃은 발생하지 않습니다!

1. **요청 타임아웃**: 3600초 (60분)로 설정되어 있음
   - 각 렌더링 작업이 60분 이내에 완료되면 타임아웃 없음

2. **대기 시간**: 
   - 인스턴스가 있으면 즉시 처리
   - 인스턴스가 없으면 새 인스턴스 생성 (몇 초 소요)
   - 최대 30개 인스턴스까지 자동 확장

3. **실제 시나리오**:
   ```
   상황: 50명이 동시에 요청
   
   - 처음 30명: 즉시 처리 시작 (각각 다른 인스턴스)
   - 나머지 20명: 대기...
   - 30명 중 1명이 완료되면 → 대기 중인 1명이 즉시 처리 시작
   - 계속 반복...
   
   결과: 모든 요청이 처리됨 (타임아웃 없음)
   ```

## Concurrency 1 vs 80 비교

### Concurrency 1 (렌더링에 권장)

**장점:**
- ✅ 안정성: 메모리 부족 위험 최소화
- ✅ 예측 가능: 각 요청이 전체 리소스 사용
- ✅ 렌더링 작업에 적합 (리소스 집약적)

**단점:**
- ❌ 인스턴스 수가 많아짐 (비용 증가)
- ❌ 최대 30개 요청만 동시 처리

### Concurrency 80 (현재 설정)

**장점:**
- ✅ 많은 요청 동시 처리 가능 (최대 2,400개)
- ✅ 인스턴스 수 절감 (비용 절감)

**단점:**
- ❌ 메모리 부족 위험 (4GB에서 80개 요청 처리 시)
- ❌ 렌더링 작업에는 부적합 (각 요청이 많은 메모리 필요)

## 권장 설정

### 렌더링 작업의 경우

**Concurrency 1 권장**

이유:
1. 렌더링은 메모리 집약적 (고해상도 이미지 처리)
2. 4GB 메모리에서 여러 요청 처리 시 메모리 부족 가능
3. 각 요청이 5-60분 소요되므로 안정성이 중요

### 설정 방법

```bash
gcloud run deploy my-project \
  --source . \
  --platform managed \
  --region asia-northeast1 \
  --allow-unauthenticated \
  --port 8080 \
  --timeout 3600 \
  --memory 4Gi \
  --cpu 2 \
  --min-instances 1 \
  --max-instances 30 \
  --concurrency 1 \  # 80 → 1로 변경
  --cpu-throttling \
  --set-env-vars GOOGLE_CLOUD_PROJECT_ID=test-ai-450613,GOOGLE_CLOUD_STORAGE_BUCKET=video-renderer-storage
```

## 결론

**Concurrency 1로 설정해도:**
- ✅ 최대 30개 요청을 동시에 처리 가능
- ✅ A수강생과 B수강생이 동시에 처리됨 (다른 인스턴스)
- ✅ 타임아웃 걱정 없음 (각 요청은 독립적으로 처리)
- ✅ 안정성 향상 (메모리 부족 위험 감소)

**31번째 요청부터는:**
- 대기하지만, 기존 요청이 끝나면 즉시 처리
- 또는 인스턴스가 추가로 생성되면 처리
- 타임아웃은 발생하지 않음 (요청 타임아웃 60분)
