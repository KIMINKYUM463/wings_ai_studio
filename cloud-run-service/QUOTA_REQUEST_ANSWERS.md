## 4. 이 서비스의 사용 사례는 무엇인가요? 예를 들어 부하 테스트, 웹 앱 제공, 일괄 처리가 있습니다.
```
영상 렌더링 서비스 (Video Rendering Service)

수강생 300명을 위한 온라인 교육 플랫폼의 일부로,
사용자가 생성한 콘텐츠(이미지, 오디오, 자막)를 결합하여
고품질 영상(1920x1080, 30fps)으로 렌더링하는 서비스입니다.

각 렌더링 작업은:
- CPU 집약적: FFmpeg를 사용한 영상 인코딩
- 메모리 집약적: 고해상도 이미지 및 오디오 처리
- 시간 소요: 영상 길이에 따라 5-60분 소요
```

## 5. 한도를 높게 설정해야 하는 기간이 어느 정도인가요? 예를 들어 사전 정의된 날짜, 무기한이 있습니다.
```
무기한 (Indefinite)

이 서비스는 지속적으로 운영되는 프로덕션 서비스이며,
수강생들이 언제든지 영상을 생성할 수 있어야 합니다.
```

## 6. 컨테이너가 동시 요청을 지원하나요? 현재 인스턴스당 동시 실행 한도는 어느 정도인가요?
```
아니요, 동시 요청을 지원하지 않습니다.

현재 설정: 인스턴스당 동시 실행 한도 1개 (concurrency: 1)

이유:
- 영상 렌더링은 CPU 및 메모리 집약적 작업입니다
- 각 인스턴스(8GB 메모리, 4 CPU)는 한 번에 하나의 렌더링 작업만
  처리하도록 설정되어 있습니다
- 동시 요청을 허용하면 메모리 부족으로 작업이 실패할 수 있습니다
```

## 7. 예상 리소스 사용 패턴을 설명해 주세요. 리소스 사용량 버스트/급증 또는 상대적으로 일정한 사용량이 예상되나요? 리소스 사용량이 급증할 것으로 예상되는 경우, 사용량에 예측 가능하거나 반복되는 패턴(예: 매일 특정 시간에 일괄 처리 작업, 특정 UTC 시간에 트래픽 증가)이 있을 것으로 예상되나요?
```
버스트/급증 패턴 (Burst/Spike Pattern)

예상 사용 패턴:
- 수강생 300명이 언제든지 영상 렌더링을 요청할 수 있습니다
- 특정 시간대(예: 저녁 시간, 주말)에 동시 요청이 급증할 수 있습니다
- 각 렌더링 작업은 5-60분 소요되므로, 여러 작업이 동시에 진행될 수 있습니다

최악의 시나리오:
- 30명의 수강생이 동시에 렌더링을 요청
- 각 요청은 8GB 메모리 + 4 CPU가 필요
- 총 필요 리소스: 30 인스턴스 × (8GB + 4 CPU) = 240GB + 120 vCPU

예측 가능한 패턴:
- 한국 시간 기준 저녁 7시-10시 (UTC 10:00-13:00)에 트래픽 증가
- 주말에 상대적으로 높은 사용량
```

## 8. 해당 인스턴스 수를 요청한 이유를 입력하세요.
```
수강생 300명을 위한 영상 렌더링 서비스의 안정적인 운영을 위해
최대 30개 인스턴스가 필요합니다.

현재 설정:
- 인스턴스당 리소스: 8GB 메모리 + 4 CPU
- 최대 인스턴스 수: 30개
- 인스턴스당 동시 실행: 1개 (concurrency: 1)

필요한 이유:
1. 동시 렌더링 요청 처리: 최대 30명의 수강생이 동시에
   영상을 생성할 수 있어야 합니다
2. 렌더링 작업 특성: 각 작업은 5-60분 소요되므로,
   여러 작업이 동시에 진행될 수 있습니다
3. 서비스 안정성: 쿼터 한계에 도달하면 새로운 요청이
   429 오류로 거부되어 사용자 경험이 저하됩니다
4. 확장성: 향후 수강생 수가 증가할 경우를 대비하여
   충분한 여유를 확보해야 합니다

현재 쿼터(200 vCPU, 400GB)로는:
- 최대 50개 인스턴스(4 CPU, 8GB)까지 가능하지만,
- 다른 서비스(my-service)도 리소스를 사용하므로
- 실제로는 30개 인스턴스 정도가 안전한 한계입니다

따라서 쿼터를 400 vCPU, 800GB로 증가시켜
안정적인 서비스 운영을 보장하고자 합니다.
```

## 9. Cloud Run의 인스턴스 한도를 어떻게 알게 되었나요?
```
서비스를 만들거나 업데이트할 때 허용된 한도를 초과하여 인스턴스 수를 설정하려고 시도함

8GB 메모리 + 4 CPU 설정으로 최대 30개 인스턴스를 배포하려고 시도했을 때
다음 오류가 발생했습니다:

"metadata.annotations[run.googleapis.com/maxScale]: Max instances must be set to 50 or fewer to set the requested total CPU.
Quota violated:
CpuAllocPerProjectRegion requested: 400000 allowed: 200000
MemAllocPerProjectRegion requested: 858993459200 allowed: 429496729600"
```
