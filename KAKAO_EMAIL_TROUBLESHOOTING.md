# 카카오 로그인 이메일 수집 오류 해결 가이드

## 문제 상황
카카오 개발자 센터에서 이메일 동의 항목을 필수 동의로 설정했는데도 "KOE205" 오류가 발생합니다.

## 확인 사항

### 1. 올바른 앱 선택 확인
**중요**: REST API 키 `29e0cf48de7d7a8fecec3d729db037d6`가 있는 앱을 선택했는지 확인하세요.

1. 카카오 개발자 센터 접속
2. **내 애플리케이션** 메뉴
3. REST API 키가 `29e0cf48de7d7a8fecec3d729db037d6`인 앱 선택
4. 앱 이름이 "마이수파코드"인지 확인

### 2. 카카오 로그인 활성화 확인
1. 카카오 로그인 > **활성화 설정** 탭
2. **카카오 로그인 활성화**가 **ON**인지 확인
3. OFF라면 ON으로 변경 후 저장

### 3. 동의 항목 재확인
1. 카카오 로그인 > **동의 항목** 메뉴
2. **개인정보** 섹션에서 **카카오계정(이메일)** 찾기
3. 상태 확인:
   - ✅ **필수 동의 [수집]** 또는 **선택 동의 [수집]** 표시되어야 함
   - ❌ **권한 없음**이면 설정 버튼 클릭하여 활성화

### 4. 동의 항목 설정 상세 확인
1. **카카오계정(이메일)** 항목의 **설정** 버튼 클릭
2. 다음 항목 확인:
   - **동의 목적**: 입력되어 있어야 함
   - **동의 단계**: "필수 동의" 또는 "선택 동의" 선택되어 있어야 함
   - **저장** 버튼 클릭하여 저장 확인

### 5. 설정 반영 대기
카카오 개발자 센터에서 설정 변경 후 **5-10분 정도** 기다려야 반영될 수 있습니다.

### 6. 브라우저 캐시 삭제
1. 브라우저 개발자 도구 열기 (F12)
2. 네트워크 탭에서 "캐시 비활성화" 체크
3. 또는 시크릿 모드에서 테스트

### 7. 다른 앱 확인
혹시 다른 앱의 설정을 보고 있는지 확인:
- "부스텍AI"라는 이름이 보이면 다른 앱일 가능성
- REST API 키가 `29e0cf48de7d7a8fecec3d729db037d6`인지 재확인

## 해결 방법

### 방법 1: 동의 항목 재설정
1. 카카오 로그인 > 동의 항목
2. 카카오계정(이메일) 항목의 **설정** 버튼 클릭
3. 동의 목적 다시 입력
4. 동의 단계 선택 (필수 동의 또는 선택 동의)
5. **저장** 버튼 클릭
6. 5-10분 대기 후 다시 테스트

### 방법 2: 카카오 로그인 재활성화
1. 카카오 로그인 > 활성화 설정
2. 카카오 로그인 활성화를 **OFF**로 변경 후 저장
3. 1분 대기
4. 다시 **ON**으로 변경 후 저장
5. 5-10분 대기 후 다시 테스트

### 방법 3: 임시로 이메일 없이 사용
현재 코드에서 `account_email` scope를 제거했으므로, 닉네임과 프로필 이미지만 수집됩니다.
이메일은 나중에 설정이 완료되면 다시 추가할 수 있습니다.

## 확인 체크리스트

- [ ] REST API 키 `29e0cf48de7d7a8fecec3d729db037d6`인 앱 선택
- [ ] 카카오 로그인 활성화가 ON
- [ ] 카카오계정(이메일) 동의 항목이 "필수 동의 [수집]" 또는 "선택 동의 [수집]"
- [ ] 동의 목적이 입력되어 있음
- [ ] 설정 저장 완료
- [ ] 5-10분 대기 완료
- [ ] 브라우저 캐시 삭제 또는 시크릿 모드 테스트

## 문의
위 방법으로도 해결되지 않으면 카카오 개발자 센터 고객지원에 문의하세요.
